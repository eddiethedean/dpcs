# SPEC.md Usage Policy

`SPEC.md` contains the full 26-chapter DPCS draft specification.

Cursor must treat `SPEC.md` as the authoritative source for:

- terminology
- architecture
- Canonical Object Model
- validation requirements
- diagnostics
- conformance
- versioning
- extensibility
- governance

If any document in this repository conflicts with `SPEC.md`, follow `SPEC.md`.

Do not invent behavior that is not grounded in `SPEC.md`.

When a section is ambiguous, implement the smallest conservative behavior and add a TODO referencing the relevant section.
